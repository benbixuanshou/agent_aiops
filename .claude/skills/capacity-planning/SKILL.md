# 容量规划技能

## When to Use

- CPU/内存告警持续时间 > 30 分钟
- 需要评估是否需要扩容
- 定期巡检报告需要资源趋势分析

## When NOT to Use

- 偶发性的短时 spike（< 5 分钟）
- 数据点不足（< 3 个时间点）

## 调用流程

1. 查 `system-metrics`，获取 CPU 和内存的时间序列
2. 计算每个时间段的：均值、峰值、增长率
3. 对比告警阈值（CPU > 80%、内存 > 85%），计算超标时长占比

## 分析维度

- **当前水位**: CPU/内存使用率峰值
- **增长趋势**: 10 分钟内是否持续上升
- **超标比例**: 超标时间 / 总监测时间
- **关联影响**: CPU 高时是否导致响应慢或 OOM

## 输出格式

```
## 容量评估

### 当前水位
| 指标 | 均值 | 峰值 | 阈值 | 超标时长 |
|-----|-----|-----|------|---------|

### 趋势判断
- CPU: 持续上升 / 稳定 / 下降
- 内存: 持续上升 / 稳定 / 下降（如有 OOM 说明内存泄漏可能）

### 扩容建议
- 当前配置: X 核 / Y GB
- 建议配置: X' 核 / Y' GB
- 理由:

### 风险预警
- 按当前增速，预计 X 小时内达到阈值
```
