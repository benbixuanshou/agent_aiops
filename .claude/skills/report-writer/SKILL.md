# 报告撰写技能

## When to Use

Agent 已完成至少一轮工具调用且证据充分，准备输出最终 Markdown 报告。

## When NOT to Use

- 还在排查中、证据不充分
- 工具调用失败需要重试

## 报告质量标准

### 必含部分
- **活跃告警清单**: 表格列出所有告警名称、级别、服务、时间、状态
- **根因分析**: 至少一个告警有具体症状+日志证据+根因结论
- **处理方案**: 至少 3 条可执行的具体步骤（含命令或操作）
- **结论**: 整体评估 + 风险评估（高/中/低）

### 禁止事项
- 编造未实际查询到的数据
- 使用"可能"、"大概"超过 3 次（应基于证据）
- 报告中出现 "tool_call"、"observation" 等内部术语

### 加分项
- 处理方案有具体命令（如 `systemctl restart xxx`）
- 评估了告警间的关联关系（A 导致 B？）
- 给出了后续监控建议

## 输出格式

从 "# 告警分析报告" 开始，使用 ## 和 ### 分层。报告末尾空一行。
