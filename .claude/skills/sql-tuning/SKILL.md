# SQL 调优技能

## When to Use

- 日志中出现 "慢查询"、"query_time"、"连接池耗尽"
- 响应时间告警（SlowResponse）且怀疑是数据库导致的
- 用户明确提到数据库性能问题

## When NOT to Use

- 慢查询日志主题不可用
- 问题明确与数据库无关（纯代码 bug、网络故障）

## 调用流程

1. 先查 `database-slow-query` 主题，拉取最近的慢查询
2. 对每条慢查询提取：执行时间、扫描行数、是否使用索引、涉及的表
3. 按表分组，找出高频慢查询的表

## 分析维度

对每条慢查询判断：
- **是否全表扫描**: 扫描行数 > 10 万 → 严重
- **是否缺索引**: index_used = "NONE" → 需建索引
- **是否结果集过大**: 考虑分页
- **是否 JOIN 过多**: 考虑拆分查询

## 输出格式

```
## 慢查询分析

### Top 慢查询
| 查询 | 执行时间 | 扫描行数 | 索引 | 表 |
|-----|---------|---------|------|-----|

### 优化建议
1. [表名] 建索引: `CREATE INDEX idx_xxx ON xxx(xxx)`
2. [表名] 加 LIMIT 分页
3. 查询拆分...

### 预期效果
- 查询 1 预计从 3.2s 降至 < 0.1s（建索引后）
```
